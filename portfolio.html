문제 정의 및 개선 (AS-IS vs TO-BE)

AS-IS (문제점)

  • 파편화된 엑셀 파일로 인한 데이터 부정합
  • 유휴 인력 현황 파악에 수일 소요
  • 계약 변경 이력 추적 불가, M/M 계산 오류
  • 수동 문서 작업으로 인한 과도한 행정 부담
  • 실투입 비용과 계약 비용의 실시간 비교 불가

TO-BE (개선)

  • Airtable 기반 중앙화된 데이터베이스 구축
  • AI 추천 로직을 통한 즉각적인 인력 매칭
  • 계약 스냅샷 기능으로 모든 변경 이력 추적
  • Make 연동으로 계약서/투입 문서 자동 생성
  • M/M 단가 연동 실시간 비용 비교 및 시뮬레이션

전체 아키텍처

Full Lifecycle 관리

Airtable을 중심으로 수주, 계약, 인력 배치, 변경 관리, 문서 자동화까지 프로젝트의 전체 생애주기를 단일 시스템에서 관리합니다.

7개 핵심 테이블

중앙화된 데이터 관리를 위해 7개의 핵심 테이블을 설계했습니다.

  • 프로젝트 기본정보
  • 계약정보 (스냅샷 포함)
  • 실제 투입 계획
  • 월별 투입 현황 (M/M)
  • 변경 작업 (Delta)
  • 인력 Pool (단가 포함)
  • 간접비 및 기타 항목

자동화 허브 (Make)

Make(Integromat)를 자동화 허브로 사용하여 Airtable의 데이터 변경을 트리거로 복잡한 비즈니스 로직과 문서 자동화를 처리합니다.

자동화 플로우 요약

1. 계약 확정 자동화

'계약 확정' 체크 시, 해당 계약의 모든 세부 내역이 스냅샷으로 생성(버전 관리)되며, 계약정보의 상태가 '확정'으로 자동 변경됩니다.

2. 실제 투입 자동화

팀장이 입력한 '실제 투입 계획'을 기반으로, 각 인력의 '월별 투입 현황(M/M)' 테이블에 데이터가 자동으로 매핑되어 생성됩니다.

3. 변경(Deltas) 자동화

'기준 라인 복제' 버튼 클릭 시, 확정된 스냅샷 라인이 '변경 작업' 테이블로 복제됩니다. 변경 확정 시 새 버전의 스냅샷이 생성되고 이전 버전은 비활성화됩니다.

4. 문서 자동화

Airtable 데이터가 Make를 통해 Google Docs 템플릿으로 전송되어 계약서, 투입 보고서 등이 PDF로 자동 생성되고, 해당 링크가 Airtable에 즉시 업데이트됩니다.

AI 인력 추천 로직 (Rule-based)

1

1차 필터링

요구 조건(분야, 등급, 스케줄)에 맞는 인력을 1차 필터링합니다.

  • 분야/등급 일치
  • 기간 내 M/M 1.0 미만 (투입 가능)
2

가산점 부여

1차 필터링 통과 인력에게 내부 기준에 따라 가산점을 부여합니다.

  • 유사 PJT 경험 (+5점)
  • 기술사 자격 (+20점)
  • PMP 자격 (+10점)
3

TOP 3 추천

합산 점수가 높은 3명을 추천하고, "기술사 보유" 등 추천 사유를 자동 생성합니다.

  • 최종 TOP 3 선정
  • 추천 사유 자동 생성

비용 관리 기능

실시간 비용 자동 계산

'인력 Pool' 테이블에 개인별 M/M 단가 필드를 추가했습니다.

  • (실제 M/M) × (개인별 단가) = 실투입 비용
  • 프로젝트별 총 투입 비용이 실시간으로 자동 집계됩니다.

비교 및 시뮬레이션

데이터 기반 의사결정을 지원합니다.

  • 계약 vs 실제: 계약 비용과 실투입 비용을 실시간으로 비교하여 예산 초과 리스크를 즉각 파악합니다.
  • 시뮬레이션: 인력 연장 또는 증원 시나리오에 따른 예상 비용을 시뮬레이션합니다.
